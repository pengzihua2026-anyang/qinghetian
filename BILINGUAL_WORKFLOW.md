# Bilingual Documentation Workflow

## Workflow Principle
**English First, Chinese Second** - All publicly released documents follow this format.

## Document Structure
```
[English Content - Complete Version]

---

[Chinese Content - Complete Version]
```

## Work Process

### Phase 1: Content Creation (Chinese)
1. **Communication**: All discussions and planning in Chinese
2. **Drafting**: Create initial document in Chinese
3. **Review**: Review and refine Chinese content

### Phase 2: Translation (English)
1. **Translation**: Translate Chinese content to English
2. **Localization**: Adapt terminology for international audience
3. **Quality Check**: Ensure translation accuracy and fluency

### Phase 3: Document Assembly
1. **English First**: Place English content at the beginning
2. **Separator**: Add `---` separator line
3. **Chinese Second**: Place Chinese content after separator
4. **Final Review**: Check bilingual consistency

### Phase 4: Publication
1. **Git Commit**: Commit with bilingual description
2. **GitHub Update**: Push to repository
3. **Verification**: Check online display

## File Naming Convention
- Use descriptive names in English
- Include Chinese translation in document content
- Example: `README.md` (contains both English and Chinese)

## Translation Guidelines

### Terminology Consistency
| English | Chinese | Notes |
|---------|---------|-------|
| QingHeTian | 青禾田 | Project name |
| Binary Editor | 二进制编辑器 | Core product |
| Zero-dependency | 零依赖 | Key feature |
| AI-driven | AI驱动 | Technology focus |

### Style Guidelines
1. **English Style**: Professional, clear, international
2. **Chinese Style**: Natural, accurate, culturally appropriate
3. **Format Consistency**: Same structure in both languages
4. **Link Consistency**: Same links work in both sections

## Quality Standards

### English Content
- Grammatically correct
- Terminology consistent
- Culturally appropriate
- Easy to understand for international audience

### Chinese Content
- Accurate translation
- Natural expression
- Culturally appropriate
- Easy to understand for Chinese audience

### Bilingual Consistency
- Same information in both languages
- Same document structure
- Same links and references
- Same update timing

## Update Process

### Minor Updates
1. Update Chinese content
2. Translate to English
3. Update bilingual document
4. Commit and push

1. Plan update in Chinese
2. Create Chinese draft
3. Translate to English
4. Bilingual review
5. Finalize and publish
